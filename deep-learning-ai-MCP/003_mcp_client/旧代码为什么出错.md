没关系，这个点确实值得彻底搞懂。

## 核心规则

Anthropic tool calling 的硬规则是：

```text
assistant message 里出现了几个 tool_use
下一条 user message 里就必须立刻包含几个对应的 tool_result
```

也就是：

```text
assistant: tool_use A, tool_use B
user:      tool_result A, tool_result B
```

不能变成：

```text
assistant: tool_use A, tool_use B
user:      tool_result A
assistant: ...
```

这样就会报：

```text
Each tool_use block must have a corresponding tool_result block in the next message.
```

---

## 正确形式

假设 Claude 一轮返回：

```text
text
tool_use extract_info #1
tool_use extract_info #2
```

正确的 `messages` 必须是：

```python
messages = [
    {
        "role": "user",
        "content": "search physics papers and extract details"
    },
    {
        "role": "assistant",
        "content": [
            {
                "type": "text",
                "text": "I found 2 papers. Let me extract details."
            },
            {
                "type": "tool_use",
                "id": "toolu_A",
                "name": "extract_info",
                "input": {"paper_id": "1910.11775v2"}
            },
            {
                "type": "tool_use",
                "id": "toolu_B",
                "name": "extract_info",
                "input": {"paper_id": "hep-ex/9605011v1"}
            }
        ]
    },
    {
        "role": "user",
        "content": [
            {
                "type": "tool_result",
                "tool_use_id": "toolu_A",
                "content": "paper A details..."
            },
            {
                "type": "tool_result",
                "tool_use_id": "toolu_B",
                "content": "paper B details..."
            }
        ]
    }
]
```

注意三点：

1. `tool_result` 的 `tool_use_id` 必须对应前面的 `tool_use.id`
2. 所有 `tool_result` 必须在**下一条 user message**
3. 如果 assistant 一次给了 2 个 `tool_use`，user 下一条 message 就要给 2 个 `tool_result`

## 正确代码怎么解决？

正确代码做了一个关键改变：

```python
tool_results = []

for content in response.content:
    if content.type == "text":
        print(content.text)

    elif content.type == "tool_use":
        result = await self.session.call_tool(...)
        tool_results.append({
            "type": "tool_result",
            "tool_use_id": content.id,
            "content": result_text,
        })

if not tool_results:
    break

messages.append({
    "role": "user",
    "content": tool_results,
})
```

重点：

```text
先遍历完整个 response.content
先执行所有 tool_use
先收集所有 tool_results
最后一次性 append 到 messages
```

不是：

```text
执行一个 tool → append 一个 tool_result → 立刻 call Claude
```

而是：

```text
执行所有 tools → append 所有 tool_results → 再 call Claude
```

## 用一句话总结错误

旧代码的问题不是 MCP，也不是 server。

问题是：

```text
它把一次 assistant response 里的多个 tool_use 拆散处理了。
```

Anthropic 要求：

```text
一个 assistant message 里的所有 tool_use
必须由下一条 user message 里的所有 tool_result 一次性回应。
```

---

## 最重要的心智模型

把一次 Claude response 当成一个整体，不要拆开中途重新调用模型。

```text
Claude response 是一个完整动作包
里面可能有：
- text
- tool_use A
- tool_use B
- tool_use C
```

你必须完整处理这个动作包：

```text
执行 A、B、C
收集 result A、B、C
一次性返回
```

然后才进入下一轮 Claude 调用。
